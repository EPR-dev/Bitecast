# Bitecast — Fiesta Island

A single-place demo that forecasts the bite at **Fiesta Island, Mission Bay (San Diego)**. Tide-aware, species-aware, offline-first, fully client-side.

**Live demo:** [https://epr-dev.github.io/Bitecast/](https://epr-dev.github.io/Bitecast/)

**Source:** [https://github.com/EPR-dev/Bitecast](https://github.com/EPR-dev/Bitecast)

Built with vanilla JS + MapLibre + a Service Worker. No backend.

---

## What it does

- **Should I go right now?** Plain-language Go / OK / Skip verdict for any moment in the next 7 days, combining NOAA tides, Open-Meteo marine weather, and Mission Bay species biology.
- **Where exactly should I cast?** A continuous species-aware heat map across a ~55 m bathymetric grid, with a "best spot" pin and a per-cell explanation (depth, distance to shore, distance to eelgrass, bottom class).
- **What's biting?** Twelve Mission Bay species, each with a habitat profile (depth gaussian + shore / eelgrass / channel preferences) and season / tide / time-of-day rules.
- **Scrub through time.** Interactive 12-hour SVG tide timeline with sun bands — drag the marker and the verdict, species recommendations, and heat map all re-score.
- **Plan up to a week ahead.** Date chip strip across the full 7-day forecast.
- **Catch journal.** IndexedDB log of catches with photo, species, length / weight, tide / wind / time-of-day snapshot. Never leaves your device.
- **Mission Bay slams.** Themed multi-species challenges (Inshore Slam, Summer Bite, Shore-to-Channel, Night Bite). Cards light up dot by dot as you check off species.
- **Discovery deck + achievements.** Pokédex-style 12-species deck plus twelve milestone badges.
- **Community big-fish board.** Anonymous handles, vague zone names only — no spot-burning, ever.
- **Field mode.** High contrast and bigger touch targets for sunlight on a boat deck.
- **Offline-first.** Service Worker pre-caches the app shell, GeoJSON, and map tiles. NOAA / Open-Meteo responses get a `last-good` cache in `localStorage` so the app never fully breaks when you lose signal.

## Stack

- [MapLibre GL JS](https://maplibre.org/) 4.7.1 for vector rendering
- Vanilla JavaScript, no build step, no `package.json`
- GeoJSON for every spatial layer
- NOAA Tides and Currents API (station `9410170` — La Jolla / Mission Bay)
- Open-Meteo for hourly marine weather
- IndexedDB for the catch journal
- A Service Worker (`sw.js`) for offline caching

## Run locally

`index.html` references local GeoJSON, so opening it as `file://` will be blocked by browser CORS. Run any static server from the project root.

```bash
# Python (any version)
python -m http.server 3456

# or Node
npx --yes serve .
```

Then open `http://localhost:3456` (or whatever port the server picks).

The first load fetches map tiles + GeoJSON; subsequent loads are served from the Service Worker cache and work offline. If you change source files during development, **hard-refresh twice** (`Ctrl+Shift+R`) — the first refresh registers the new Service Worker, the second picks up the fresh assets.

## Data accuracy

Every coordinate in `data/access.geojson` and `data/pois.geojson` was audited against the Fiesta Island shore polygon (`scripts/_audit_points.mjs`). Six points were systematically mis-projected by the initial OSM pull (~300 m offset into the East Channel) and were hand-corrected. Hazards and eelgrass beds are intentionally in water — that's where hazards and eelgrass belong. The boat launch sits on the adjacent Vacation Isle, which is correct.

The bathymetric grid (`data/bathy_grid.geojson`) is generated by `scripts/bake_bathy_grid.mjs` using inverse-distance weighting from hand-placed depth control points and clipped to water within 550 m of the Fiesta Island shore.

## Design thesis

Anyone can render a tide chart. The value compounds in the **place knowledge** — hand-curated bathymetry, species habitat profiles, and per-place rules that turn raw data into a verdict a 14-year-old can act on. See the architecture canvas (or the README of the next region) for the scale-up path.
